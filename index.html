Drone Simulation

Developed in the course of Program Design & Development in a group of 4, this project is coded in C++ with a combination of HTML and Javascript to perform a simulation of a delivery system with drones.

Entities

This simulation consists of 3 different entities, Drone, Package and Customer, each package is assigned to a customer. The idea is that multiple information of entities are loaded in as a SQL file, created with a composite factory pattern, then the drones will calculate the path to the closest package and traverse to pick up the package and deliver to the customer.

Activity Logs

While the entities are moving to perform their assigned task, their activities are also recorded using the Observer pattern, which also acts as a notification in the system to manage the delivery status.




Battery Tracking

Each drone has a certain amount of battery charge that dissipates while moving. When the battery charge goes to 0, the drone will stop moving and drop the package at its location. Then the closest live drone will come by to pick up the dropped package to finish the task. Using the Decorator pattern, we are able to color code the amount of battery charge of each drone during the simulation.

Paths

The drones can have different movements based on what is assigned in the SQL file, the Beeline, Parabolic and Smart. The Beeline path makes the drone rise to a set altitude while moving 45 degree upwards, move in a straight line towards the package and descend at a 45 degree angle when it reaches the destination. Parabolic path makes the drone rise and move towards the destination in a parabola. The smart path uses A* search algorithm to find the path to get to the destination on the simulated map.

Spectrogram

Spectrogram is a visualization of signal strength over time at different frequencies in a waveform. In the course of Natural Language Processing, I used python to create a Spectrogram of wav files from the TIMIT corpus. This process includes reading the signals in binary, translating the signals to integers and applying a hamming window function then using fourier transform to find strengths in each signal. Finally convert the data into a relative grayscale to create an Spectrogram image of that particular audio file.

Database design

With a group of 4 in the Course of Database Systems, we designed a database for a hypothetical clinic. With a list of conditions, we first draw out the ER diagram for the system, then mapping its relations. Lastly, we implemented them into a SQL database and tested them with SQL queries.

CYK Parser with Context Free Grammar

In the Natural Language Processing course, I made a program with python to distinguish a sentence's "meaning" using Cocke–Younger–Kasami(CYK, or CKY) algorithm, which is a parsing algorithm for context free grammar(CFG). This algorithm is able to take a sentence, determine if it is valid for the given CFG, or parse all possible derivations of the sentence based on the CFG.

Inventory tracking app in iOS

During a course of Mobile App Development in high school, I created an iOS application designed specifically for tracking art supplies in a classroom setting. The idea is for students to be able to borrow supplies from an automatic system that keeps a log on who, when and how many items did the students borrow, and notify the teacher when items need to be returned by the students.
